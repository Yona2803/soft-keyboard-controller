⌨️ Soft Keyboard Controller

Interactive playground for every option and method in the README.

keyboard: disabled activation: dblclick

1. Configuration

Change any option below, then click Apply & Reinitialize to rebuild the controller with a fresh new SoftKeyboardController(options) call — exactly like the README's "Configuration" example.

Reinitializing calls destroy() on the previous instance first, then constructs a new one — so you can freely experiment with every combination.

2. Public API

These buttons call the controller's public methods directly on the live instance.

keyboard.enabled → false

3. Test inputs

Each field shows its live inputmode attribute. Try double-clicking / double-tapping / clicking (depending on your chosen activation), then click elsewhere to see deactivateOnBlur in action.

4. Dynamic inputs

Adds a brand-new <input> to the DOM and calls refresh(), exactly like the README's "Dynamic Inputs" example.

5. Function-based ignore

Toggle this to swap the ignore option from a CSS selector to a function, matching the README's ignore(element) example (element.dataset.keyboard === "false").

6. Activity log

Fires from the onEnable / onDisable callbacks and every API call you trigger above.
